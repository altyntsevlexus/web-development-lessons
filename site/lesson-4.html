CSS · Урок 04

Display та Позиціонування

Як елементи розташовуються на сторінці: потік документа, position, z-index, display, float
Теорія Предмет: Веб Клас: 7 ~45 хв
01

Потік документа

За замовчуванням браузер розміщує елементи зверху вниз у порядку їх появи в HTML. Це називається нормальним потоком (normal flow).

Блокові елементи (<div>, <p>, <h1>) займають весь рядок і стоять один під одним. Рядкові елементи (<span>, <a>) течуть як слова в тексті.

┌──────────────────────────────────┐ │ <div> блок — займає весь рядок │ ├──────────────────────────────────┤ │ <div> наступний блок │ ├──────────────────────────────────┤ │ <span> <span> рядкові елементи │ └──────────────────────────────────┘

Властивості position та display дозволяють змінити цей потік і розмістити елементи саме так, як потрібно.

02

position: relative

Елемент залишається в потоці, але може бути зсунутий відносно свого нормального місця за допомогою top, left, right, bottom.

.card {
  position: relative;
  top:      10px;  /* зсув вниз на 10px */
  left:     20px;  /* зсув вправо на 20px */
}
💡 ГОЛОВНА РОЛЬ
Найчастіше position: relative використовують не для зсуву, а щоб зробити елемент «контейнером позиціонування» для вкладених absolute-елементів.

Місце, яке елемент займав у потоці, зберігається — інші елементи не «заповнюють» його позицію.

03

position: absolute

Елемент повністю виходить з потоку і позиціонується відносно найближчого предка з position (не static). Якщо такого предка немає — відносно вікна браузера.

.parent {
  position: relative; /* контейнер */
}

.badge {
  position: absolute;
  top:      16px;
  right:    16px; /* кут батьківського елемента */
}
┌─────────────────────┐ │ .parent (relative) │ [.badge] ←─ absolute: │ │ top: 16px, right: 16px │ ┌───────┐ │ │ │ badge │ │ │ └───────┘ │ └─────────────────────┘
⚠ ВАЖЛИВО
Absolute-елемент не займає місця в потоці — решта елементів його ігнорують і можуть перекриватися з ним.
04

position: fixed

Елемент фіксується відносно вікна браузера (viewport). Він не рухається при прокрутці.

.header {
  position: fixed;
  top:      0;
  width:    100%;
  z-index:  100;
}

Типові застосування: навігаційна шапка, кнопка «вгору», чат-кнопка, cookie-банер.

💡 КОМПЕНСАЦІЯ
Fixed-елемент виходить з потоку. Щоб вміст не ховався під шапкою, додайте padding-top до body — рівний висоті шапки.
05

position: sticky

Гібрид relative і fixed. Елемент рухається в потоці як звичайний, але «прилипає» до краю viewport при прокрутці до заданого порогу.

.sidebar {
  position: sticky;
  top:      80px; /* прилипає коли залишається 80px до верху */
}
До прокрутки: Під час прокрутки: ┌──────────────┐ ┌──────────────┐ │ header │ │ header │ ← fixed ├──────────────┤ ├──────────────┤ │ content │ │ sidebar │ ← sticky (прилип!) │ sidebar │ │ │ │ │ │ content │ └──────────────┘ └──────────────┘
⚠ ОБМЕЖЕННЯ
Sticky не працює, якщо батьківський елемент має overflow: hidden або overflow: auto.
06

z-index — порядок шарів

Коли елементи перекриваються, z-index визначає, хто буде «зверху». Більший номер — вище.

.header {
  position: fixed;
  z-index:  100;
}

.popup {
  position: absolute;
  z-index:  200; /* поверх header */
}

.overlay {
  position: fixed;
  z-index:  150; /* між header і popup */
}
⚠ УМОВА
z-index працює лише на позиціонованих елементах — тих, що мають position, відмінний від static (тобто relative, absolute, fixed або sticky).
💡 КОНВЕНЦІЯ
У реальних проектах задають «рівні»: контент — 1–10, шапка — 100, модальне вікно — 1000, tooltip — 9999.
07

display: inline

Елемент ведеться як частина тексту: не починає новий рядок і займає рівно стільки ширини, скільки потрібно для вмісту.

span    { display: inline; }   /* за замовчуванням */
a       { display: inline; }
strong  { display: inline; }

Обмеження display: inline:

  • width та height — не мають ефекту
  • Вертикальні margin (top/bottom) — не впливають на макет
  • Горизонтальні padding і margin — працюють
💡 INLINE ЗА ЗАМОВЧУВАННЯМ
<span>, <a>, <strong>, <em>, <img>, <button>, <input>
08

display: block

Елемент починає новий рядок і займає всю доступну ширину батьківського контейнера.

div { display: block; }   /* за замовчуванням */
p   { display: block; }
h1  { display: block; }

Переваги display: block:

  • Можна задати width, height
  • Всі padding та margin (у тому числі top/bottom) — працюють
  • margin: 0 auto — центрує елемент горизонтально
💡 BLOCK ЗА ЗАМОВЧУВАННЯМ
<div>, <p>, <h1>–<h6>, <section>, <article>, <ul>, <li>
09

display: inline-block

Гібрид: елементи стоять в рядок (як inline), але їм можна задавати ширину, висоту і всі відступи (як block).

.button {
  display:        inline-block;
  width:          120px;
  height:         40px;
  padding:        8px 16px;
  vertical-align: top; /* вирівнювання по верху */
}
display: block display: inline-block ┌──────────────────┐ ┌────┐ ┌────┐ ┌────┐ │ елемент │ │ │ │ │ │ │ ├──────────────────┤ └────┘ └────┘ └────┘ │ елемент │ └──────────────────┘

Сьогодні inline-block часто замінюють flex, але він досі корисний для кнопок, іконок, чіпів.

10

Підсумкова таблиця

Властивість Значення Ефект У потоці?
position relative Зсув від свого місця; контейнер для absolute Так
position absolute Відносно найближчого positioned предка Ні
position fixed Фіксований відносно вікна браузера Ні
position sticky Прилипає при прокрутці до порогу Так / Ні*
z-index число Порядок шарів (більше = вище) —
display inline Рядковий; без width/height Так
display block Новий рядок; повна ширина Так
display inline-block Рядковий + підтримує width/height Так
