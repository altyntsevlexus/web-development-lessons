HTML · Урок 02

Стандарти HTML-документа

DOCTYPE, структура документа, head, title, посилання, пробіли, відступи та коментарі
Теорія ~30 хв
01

<!DOCTYPE html>

Перший рядок будь-якого HTML-файлу — це оголошення типу документа. Воно повідомляє браузеру, що файл написаний за стандартом HTML5.

<!DOCTYPE html>
<html>
  ...
</html>
⚠ ЗАВЖДИ ПЕРШИМ
<!DOCTYPE html> повинен бути першим рядком файлу — до нього нічого не має бути, навіть порожніх рядків. Без нього браузер може перейти в режим сумісності і відображати сторінку некоректно.
02

Структура HTML-документа

Кожен HTML-файл має стандартну структуру з трьох основних частин:

<!DOCTYPE html>
<html lang="uk">

  <head>
    <!-- інформація про сторінку -->
  </head>

  <body>
    <!-- видимий вміст -->
  </body>

</html>
<!DOCTYPE html>
<html>
<head> — невидима частина: мета-дані, заголовок вкладки, підключення CSS
<body> — видима частина: весь контент сторінки
</html>

Атрибут lang="uk" вказує браузеру і пошуковим системам мову вмісту сторінки.

03

<head> — інформація про сторінку

Тег <head> містить мета-дані: інформацію про сторінку, яку браузер використовує, але не показує користувачу.

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Назва вкладки</title>
  <link rel="stylesheet" href="style.css">
</head>
  • charset="UTF-8" — підтримка кирилиці та інших символів
  • viewport — коректне відображення на мобільних
  • <title> — текст у вкладці браузера
  • <link> — підключення CSS-файлу
04

<title> — назва вкладки

Тег <title> всередині <head> задає текст, який з'являється у вкладці браузера і в результатах пошуку Google.

<head>
  <title>Мій перший сайт</title>
</head>
Де з'являється title
Мій перший сайт ×
+
← → ⟳ 🔒 example.com
💡 SEO
Хороший title — коротко і точно описує зміст сторінки (до 60 символів). Пошукові системи показують його у результатах пошуку.
05

Посилання <a>

Тег <a> (anchor — якір) створює посилання. Атрибут href вказує куди воно веде.

<!-- Зовнішнє посилання -->
<a href="https://google.com">Відкрити Google</a>

<!-- Внутрішня сторінка -->
<a href="about.html">Про нас</a>

<!-- Відкрити в новій вкладці -->
<a href="https://google.com" target="_blank">Google</a>
  • Зовнішні посилання починаються з https://
  • Внутрішні — просто ім'я файлу: about.html
  • target="_blank" — відкрити в новій вкладці
06

Якоря — переходи по сторінці

Можна створити посилання на конкретний розділ тієї самої сторінки. Для цього елементу присвоюється id, а посилання веде на #id.

<!-- Посилання-якір (зазвичай у навігації) -->
<a href="#contacts">Перейти до контактів</a>

<!-- ... далі по сторінці ... -->

<!-- Цільовий розділ -->
<section id="contacts">
  <h2>Контакти</h2>
  ...
</section>
💡 ЗАСТОСУВАННЯ
Якоря використовують для навігаційних меню на landing-сторінках: «Про нас», «Послуги», «Контакти» — кліки прокручують сторінку до потрібного розділу.
07

Пробіли і відступи в коді

Браузер ігнорує зайві пробіли, переноси рядків і відступи в HTML-коді — вони впливають лише на читабельність коду, не на вигляд сторінки.

<!-- Обидва варіанти дають однаковий результат: -->

<p>Привіт</p><p>Світ</p>

<p>Привіт</p>
<p>Світ</p>

Між словами в тексті кілька пробілів підряд браузер стиснить до одного. Для реальних відступів між елементами використовуй CSS margin.

✓ ПОРАДА
Залишай порожні рядки між великими секціями коду — так код легше читати.
08

Відступи (індентація)

Вкладені елементи прийнято зсувати вправо на два або чотири пробіли (або один Tab). Це не впливає на роботу коду, але робить структуру наочною.

<!-- Погано — важко зрозуміти структуру -->
<ul><li>HTML</li><li>CSS</li></ul>

<!-- Добре — структура видна одразу -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
</ul>
💡 ПРАВИЛО
Кожен рівень вкладеності — один відступ Tab. Відступи показують відносини батько–дитина так само, як дерево структури.
09

Коментарі <!-- -->

Коментарі — це нотатки в коді. Браузер їх не відображає. Вони допомагають пояснити код собі та іншим розробникам.

<!-- Це коментар — браузер його ігнорує -->

<header>
  <!-- Навігація -->
  <nav>...</nav>
</header>

<!-- TODO: додати форму зворотнього зв'язку -->
⚠ УВАГА
Коментарі видно у вихідному коді сторінки (через «Переглянути код» у браузері). Не пиши в коментарях паролі чи конфіденційні дані.
