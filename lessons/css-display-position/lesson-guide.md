# CSS: Display та Позиціонування — Конспект уроку (тільки для вчителя)

## Огляд

Учні працюють з файлами `index.html` + `style.css`. Сторінка — портал технологічних новин **TechDaily**. Кожне завдання вводить один принцип позиціонування або відображення і одразу дає видимий результат.

**Теми:**
1. `position: relative` — відносне позиціонування
2. `position: absolute` — абсолютне позиціонування
3. `position: fixed` — фіксоване позиціонування
4. `position: sticky` — «липке» позиціонування
5. `z-index` — порядок шарів
6. `display: inline` — рядкові елементи
7. `display: block` — блокові елементи
8. `display: inline-block` — гібрид
9. `float` — обтікання
10. `clear` — скасування обтікання

---

## Завдання 1 — position: relative

**Концепція:** `position: relative` переміщує елемент відносно його нормального місця в потоці. Але найважливіше — він стає **контейнером позиціонування** для вкладених `absolute`-елементів.

**Що учні бачать до:** Секція `.hero-section` лежить у нормальному потоці. Значок `BREAKING` (`.breaking-badge`) є звичайним flex-елементом внизу hero.
**Що учні бачають після:** Видимих змін майже немає — але тепер hero є **точкою відліку** для наступного завдання.

**CSS:**
```css
.hero-section {
  position: relative;
}
```

**Що обговорити:**
- Без `position: relative` (або `absolute`/`fixed`/`sticky`) елемент — «статичний» (static). Z-index і зміщення top/left/right/bottom не мають ефекту на static.
- Запитайте: *Чому ми додаємо position: relative, якщо нічого не змінилося?* (Підготовка для завдання 2.)

---

## Завдання 2 — position: absolute

**Концепція:** Елемент **виривається з потоку** і позиціонується відносно найближчого предка з `position` (не static). Якщо такого немає — відносно вікна браузера.

**Що учні бачать до:** Значок `BREAKING` знаходиться внизу hero разом з текстом (як flex-елемент).
**Що учні бачають після:** Значок «вистрибує» в лівий верхній кут hero-секції — прямо як у справжньому новинному сайті.

**CSS:**
```css
.breaking-badge {
  position: absolute;
  top: 20px;
  left: 20px;
}
```

**Що обговорити:**
- Елемент з `position: absolute` **не займає місця** в потоці — решта елементів його не «бачать».
- Прив'язка: тому hero (завдання 1) мав `position: relative` — badge позиціонується саме відносно нього.
- Запитайте: *Що станеться, якщо прибрати position: relative з hero?* (Badge полетить відносно сторінки або вікна.)

---

## Завдання 3 — position: fixed

**Концепція:** Елемент фіксується відносно **вікна браузера** (viewport). Він не рухається при прокрутці.

**Що учні бачать до:** Шапка сайту прокручується разом зі сторінкою і зникає вгорі.
**Що учні бачають після:** Шапка залишається на місці при прокрутці — стандартна поведінка навігації реальних сайтів.

**CSS:**
```css
.site-header {
  position: fixed;
  top: 0;
  width: 100%;
}
```

**Що обговорити:**
- `padding-top: 80px` вже встановлено на `body` — воно компенсує фіксовану шапку, щоб вміст не «заховався» за нею.
- Fixed-елемент також **виходить з потоку** — саме тому потрібен padding на body.
- Запитайте: *Де ще на сайтах зустрічається position: fixed?* (Чат-кнопки, cookie-банери, бічні меню.)

---

## Завдання 4 — position: sticky

**Концепція:** Гібрид `relative` і `fixed`. Елемент рухається з потоком, але **«прилипає»** при досягненні заданого порогу прокрутки.

**Що учні бачать до:** Бічна панель (`.sidebar`) прокручується разом зі сторінкою і зникає вгорі.
**Що учні бачають після:** При прокрутці sidebar зупиняється на відстані 80px від верху вікна і залишається видимим, поки є вміст поруч.

**CSS:**
```css
.sidebar {
  position: sticky;
  top: 80px;
}
```

**Що обговорити:**
- `top: 80px` — відстань від верху viewport, де sidebar «прилипає» (80px = висота шапки).
- Для роботи sticky батьківський елемент НЕ повинен мати `overflow: hidden`.
- Запитайте: *Де на сайтах зустрічається sticky?* (Бічне меню, таблиця з заголовком, nav всередині секції.)

---

## Завдання 5 — z-index

**Концепція:** Контролює порядок шарів (хто «зверху») для позиціонованих елементів. Більший номер — вище.

**Що учні бачать до:** Червоний значок «В ПРЯМОМУ ЕФІРІ» (`.floating-alert`) перекриває навігацію в шапці, бо він DOM-пізніший і має вищий пріоритет у стеку.
**Що учні бачають після:** Шапка отримує `z-index: 100` і виходить на перший план — значок тепер під шапкою.

**CSS:**
```css
.site-header {
  z-index: 100;
}
```

**Що обговорити:**
- `z-index` працює **лише на позиціонованих** елементах (не static).
- `.floating-alert` вже має `position: fixed` у базових стилях.
- Запитайте: *Яке значення z-index дати шапці, щоб вона була НИЖЧЕ значка?* (Менше за z-index значка, наприклад 1.)

---

## Завдання 6 — display: inline

**Концепція:** `display: inline` — елемент ведеться як частина тексту. Не починає новий рядок, ширину/висоту встановити не можна.

**Що учні бачать до:** Теги-чіпи (`.topic-chip`) є `<div>` — блоковими елементами. Кожен займає весь рядок, теги стоять у стовпчик.
**Що учні бачають після:** Усі теги стоять в один рядок як слова в тексті — виглядають як справжні чіпи.

**CSS:**
```css
.topic-chip {
  display: inline;
}
```

**Що обговорити:**
- `<span>` за замовчуванням — `display: inline`. `<div>` — `display: block`.
- `display: inline` ігнорує `width` і `height`. Якщо потрібні розміри — використовуйте `display: inline-block`.
- Запитайте: *Які HTML-теги за замовчуванням inline?* (`<a>`, `<span>`, `<strong>`, `<em>`, `<img>`.)

---

## Завдання 7 — display: block

**Концепція:** `display: block` — елемент займає всю ширину рядка і починає новий рядок. Можна задавати width, height, padding, margin.

**Що учні бачать до:** Посилання `.footer-link` — це `<a>`, тобто inline за замовчуванням. Всі посилання стоять в один рядок.
**Що учні бачають після:** Кожне посилання займає свій рядок — стовпчик навігації у підвалі.

**CSS:**
```css
.footer-link {
  display: block;
  padding: 6px 0;
}
```

**Що обговорити:**
- `<a>` за замовчуванням inline, але в меню нам зручніше мати block для відступів.
- Запитайте: *Як зробити, щоб посилання займало всю ширину і мало зону кліку на весь рядок?* (display: block.)

---

## Завдання 8 — display: inline-block

**Концепція:** Гібрид: елементи стоять в рядок (як inline), але їм можна задати ширину і висоту (як block).

**Що учні бачать до:** Блоки статистики (`.stat-box`) — div-елементи, кожен займає весь рядок. Три блоки стоять один під одним.
**Що учні бачають після:** Три блоки стоять поруч, кожен з фіксованою шириною — схоже на dashboard.

**CSS:**
```css
.stat-box {
  display: inline-block;
  width: 180px;
  vertical-align: top;
  margin-right: 8px;
  margin-bottom: 0;
}
```

**Що обговорити:**
- `vertical-align: top` вирівнює блоки по верхньому краю (інакше можуть «з'їхати»).
- Порівняйте: `inline` — не можна задати width. `block` — займає весь рядок. `inline-block` — поєднує обидва.
- Запитайте: *Чим inline-block відрізняється від flex?* (Flex — для контейнера, inline-block — для самого елемента.)

## Типові помилки учнів

1. **`position: absolute` без позиціонованого батька** — елемент «летить» відносно всієї сторінки. Завжди питайте: *Що є батьком? Чи є у нього position?*
2. **`z-index` на static-елементі** — не працює. Z-index потребує `position: relative/absolute/fixed/sticky`.
3. **`position: sticky` і `overflow: hidden` на батьку** — sticky перестає працювати. Прихована помилка.
4. **`display: inline` замість `inline-block`** — якщо потрібна ширина або висота, inline не підійде.
