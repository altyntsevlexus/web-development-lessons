# CSS: Візуальні правила — Конспект уроку (тільки для вчителя)

## Огляд

Учні працюють з файлами `index.html` + `style.css`. Сторінка — сайт музичного гурту SoundWave. Кожне завдання вводить одну візуальну властивість CSS, яка одразу дає помітний результат.

**Теми:**
1. `font-family` — шрифт
2. `font-size` — розмір шрифту
3. `font-weight` — жирність
4. `text-align` — вирівнювання тексту
5. `color` + `background-color` — колір тексту і фону
6. `opacity` — прозорість
7. `background-image` — фонове зображення

---

## Завдання 1 — font-family

**Концепція:** Змінює шрифт тексту. Якщо назва з кількох слів — у лапках. Стандартний шрифт браузера — Times New Roman.

**Що учні бачать до:** Заголовки стандартним шрифтом браузера.
**Що учні бачають після:** Заголовки виглядають по-іншому — більш елегантно.

**CSS:**
```css
h1, h2 {
  font-family: 'Georgia', serif;
}
```

**Що обговорити:**
- `serif` — запасний шрифт, якщо Georgia не знайдена.
- Для веб-шрифтів (наприклад, Google Fonts) потрібне підключення через `<link>`.
- Запитайте: *Чому назви шрифтів з кількох слів беруться в лапки?* (Щоб браузер розумів, що це одна назва.)

---

## Завдання 2 — font-size

**Концепція:** Задає розмір тексту. Пікселі (`px`) — точний фіксований розмір. `rem` — відносно кореневого елемента (зазвичай 16px = 1rem).

**Що учні бачать до:** Заголовок `hero-title` стандартного розміру — непомітний на великому фоні.
**Що учні бачають після:** Великий заголовок, який домінує в hero-секції.

**CSS:**
```css
.hero-title {
  font-size: 3rem;
}
```

**Що обговорити:**
- 3rem = 48px при стандартних налаштуваннях браузера.
- Запропонуйте учням спробувати `4rem`, `5rem` — щоб відчути різницю.
- `font-size: 1em` = розмір шрифту батьківського елемента.

---

## Завдання 3 — font-weight

**Концепція:** Задає товщину шрифту. `bold` = 700, `normal` = 400. Також можна використовувати числа від 100 до 900.

**Що учні бачать до:** Посилання в навігації — звичайної товщини.
**Що учні бачають після:** Посилання виглядають чіткіше і помітніше.

**CSS:**
```css
.nav-link {
  font-weight: bold;
}
```

**Що обговорити:**
- Не всі шрифти підтримують кожне числове значення — тоді браузер обирає найближче доступне.
- Запитайте: *Яка різниця між `font-weight: 400` і `font-weight: bold`?* (Ніякої — `bold` = `700`.)

---

## Завдання 4 — text-align

**Концепція:** Вирівнювання тексту всередині елемента. Працює лише для блокових елементів.

**Що учні бачать до:** Текст у hero-overlay — притиснутий до лівого краю.
**Що учні бачають після:** Текст відцентрований — виглядає як справжній банер.

**CSS:**
```css
.hero-overlay {
  text-align: center;
}
```

**Що обговорити:**
- Значення: `left` (за замовчуванням), `center`, `right`, `justify`.
- `text-align` центрує текст *всередині* елемента, а не сам елемент на сторінці. Для центрування елемента — `margin: 0 auto`.

---

## Завдання 5 — color + background-color

**Концепція:** `color` — колір тексту. `background-color` — колір фону. Кольори можна задавати назвою, HEX, RGB або RGBA.

**Що учні бачать до:** Шапка сайту — біла з чорним текстом (стандарт браузера).
**Що учні бачають після:** Темна шапка зі світлим текстом — сайт набуває характеру.

**CSS:**
```css
.site-header {
  background-color: #1a1a2e;
  color: #ffffff;
}
```

**Що обговорити:**
- Запропонуйте спробувати різні кольори, зокрема RGBA: `rgba(26, 26, 46, 0.9)`.
- Поясніть різницю між `color` (тільки текст) і `background-color` (весь фон елемента).

---

## Завдання 6 — opacity

**Концепція:** Прозорість всього елемента від 0 (невидимий) до 1 (непрозорий). Впливає на елемент разом з усім вмістом.

**Що учні бачать до:** Hero-overlay — непрозорий блок, зображення буде закрите.
**Що учні бачають після:** Через overlay просвічує фонове зображення (якщо вже додано з завдання 7 або навпаки).

**CSS:**
```css
.hero-overlay {
  opacity: 0.85;
}
```

**Що обговорити:**
- Попросіть спробувати `opacity: 0.3`, `opacity: 0.6`, `opacity: 1` — щоб відчути шкалу.
- Різниця між `opacity: 0` і `display: none` — при `opacity: 0` елемент займає місце.
- Якщо потрібна прозорість тільки фону (а не тексту) — використовуйте `background-color: rgba(...)`.

**Порада:** Це завдання краще виконувати після завдання 7 (background-image), щоб ефект прозорості був очевидним.

---

## Завдання 7 — background-image

**Концепція:** Задає зображення як фон елемента. Завжди поєднується з `background-size` і `background-position`.

**Що учні бачать до:** Hero-секція — суцільний темний прямокутник.
**Що учні бачають після:** За текстом видно фото концерту — сторінка ожила.

**CSS:**
```css
.hero {
  background-image: url('https://images.unsplash.com/photo-1511671782779-c97d3d27a1d4?w=1200&q=80');
  background-size: cover;
  background-position: center;
}
```

**Що обговорити:**
- `background-size: cover` — заповнює весь елемент, обрізаючи краї якщо потрібно.
- `background-size: contain` — показує все зображення, але можуть бути порожні краї.
- Запропонуйте знайти інше фото на Unsplash і замінити URL.

**Порада:** Виконайте це завдання перед завданням 6, щоб учні відразу побачили ефект прозорості overlay на зображенні.

## Рекомендований порядок виконання

Через те, що завдання 6 і 7 взаємодіють, рекомендований порядок:
**1 → 2 → 3 → 4 → 5 → 7 → 6**

Спочатку додайте фонове зображення (завд. 7), а потім показуйте прозорість overlay (завд. 6) — так ефект буде найбільш наочним.

## Типові помилки учнів

1. **Лапки у `font-family`** — `font-family: Courier New` замість `font-family: "Courier New"`. Без лапок браузер читає лише перше слово.
2. **Одиниці у `font-size`** — `font-size: 18` замість `font-size: 18px`. Без одиниці CSS ігнорує значення.
3. **`opacity` vs `rgba`** — `opacity: 0.5` робить напівпрозорим і фон, і текст. Якщо текст повинен бути чітким — використовуйте `background-color: rgba(0,0,0,0.5)`.
4. **`background-image` без `background-size`** — зображення відображається мозаїкою або у своєму природному розмірі.
5. **Плутанина `color` і `background-color`** — `color` тільки для тексту.
