# CSS: Селектори — Конспект уроку (тільки для вчителя)

## Огляд

Учні працюють з файлами `index.html` + `style.css`. Сторінка — блог про відеоігри. Кожне завдання вводить новий тип селектора і одразу показує видимий результат.

**Теми:**
1. Універсальний селектор `*`
2. Селектор за тегом
3. Селектор за ID `#`
4. Селектор за класом `.`
5. Псевдоклас `:hover`
6. Ланцюжок селекторів
7. Вкладені елементи (нащадок)
8. Групування селекторів
9. Специфічність (спостереження)

---

## Завдання 1 — Універсальний селектор `*`

**Концепція:** Браузер додає власні відступи до елементів. Скидання починається з `* { margin: 0; padding: 0; }`.

**Що учні бачать до:** Елементи мають браузерні відступи — білі поля навколо сторінки, відступи між заголовками.
**Що учні бачать після:** Відступи зникають, сторінка «стискається» до країв.

**CSS:**
```css
* {
  margin: 0;
  padding: 0;
}
```

**Що обговорити:**
- `*` вибирає **кожен** елемент.
- Специфічність `*` = 0 — будь-яке інше правило перевизначить його.

---

## Завдання 2 — Селектор за тегом

**Концепція:** Назва тегу без дужок. Застосовується до всіх елементів з цим тегом.

**Що учні бачать до:** Заголовки `h2` — чорний текст за замовчуванням.
**Що учні бачають після:** Всі `h2` стали темно-синіми.

**CSS:**
```css
h2 {
  color: #0f3460;
}
```

**Що обговорити:**
- Зміна поширюється на **всі** `h2` сторінки — і в `article`, і в `aside`.
- Запитайте: *Як зробити, щоб тільки один заголовок мав інший колір?* (→ клас або ID)

---

## Завдання 3 — Селектор за ID

**Концепція:** `#` перед назвою ID. Найвища специфічність після `!important`.

**Що учні бачать до:** Назва блогу «GameZone Blog» — стандартний чорний заголовок.
**Що учні бачають після:** Великий яскравий заголовок.

**CSS:**
```css
#site-title {
  font-size: 2.5rem;
  color: #e94560;
}
```

**Що обговорити:**
- ID має бути унікальним на сторінці.
- Специфічність: `#site-title` (100) > `.header-title` (10) > `h1` (1).

---

## Завдання 4 — Селектор за класом

**Концепція:** `.` перед назвою класу. Клас може бути у багатьох елементів.

**Що учні бачать до:** Бейджики `.game-tag` — звичайний текст поруч.
**Що учні бачають після:** Яскраві кольорові таблетки-теги.

**CSS:**
```css
.game-tag {
  background-color: #e94560;
  color: white;
  padding: 4px 12px;
  border-radius: 20px;
  font-size: 0.8rem;
}
```

**Що обговорити:**
- Один клас → стиль застосовується до всіх бейджиків одразу.
- Порівняйте: якби кожен тег мав окремий ID — прийшлося б писати правило для кожного.

---

## Завдання 5 — Псевдоклас `:hover`

**Концепція:** `:hover` — стан елемента (коли на нього наведено курсор). Записується через двокрапку.

**Що учні бачать до:** Посилання — сині, підкреслені (стандарт браузера).
**Що учні бачають після:** При наведенні посилання змінює колір.

**CSS:**
```css
a:hover {
  color: #e94560;
  text-decoration: none;
}
```

**Що обговорити:**
- Псевдокласи описують **стан** або **позицію** елемента.
- Інші псевдокласи: `:focus`, `:active`, `:first-child`, `:nth-child(n)`.

---

## Завдання 6 — Ланцюжок селекторів

**Концепція:** Без пробілу між двома селекторами — елемент повинен відповідати **обом одночасно**.

**Що учні бачать до:** `h2.featured` виглядає як звичайний `h2`.
**Що учні бачають після:** Тільки заголовок із класом `featured` отримав особливий стиль.

**CSS:**
```css
h2.featured {
  color: #e94560;
  font-style: italic;
}
```

**Що обговорити:**
- `h2.featured` ≠ `.featured` — перший вибирає тільки `h2` з класом `featured`, другий — будь-який елемент із цим класом.
- Специфічність: `h2.featured` = 1 (тег) + 10 (клас) = **11**.

---

## Завдання 7 — Вкладені елементи (нащадок)

**Концепція:** Пробіл між селекторами — другий елемент знаходиться **всередині** першого.

**Що учні бачать до:** Посилання в `nav` — сині, підкреслені, стоять впритул.
**Що учні бачають після:** Посилання в навігації без підкреслення, з рівними відступами.

**CSS:**
```css
nav.site-nav a {
  text-decoration: none;
  color: #e0e0e0;
  padding: 0 12px;
}
```

**Що обговорити:**
- `nav a` вибирає **всі** `<a>` всередині `<nav>`, на будь-якій глибині вкладеності.
- Це не впливає на посилання в `aside` або `footer`.

---

## Завдання 8 — Групування селекторів

**Концепція:** Кома між селекторами — одне правило для кількох елементів одночасно.

**Що учні бачають до:** Різні шрифти у `h1`, `h2`, `h3`.
**Що учні бачають після:** Єдиний шрифт для всіх заголовків.

**CSS:**
```css
h1,
h2,
h3 {
  font-family: Georgia, serif;
}
```

**Що обговорити:**
- Це не нова властивість — просто скорочення: замість трьох окремих правил — одне.
- Кожен селектор в групі зберігає власну специфічність (h1 = 1, h2 = 1).

---

## Завдання 9 — Специфічність (спостереження)

**Концепція:** Коли кілька правил суперечать одне одному — перемагає те, що має вищу специфічність.

**Що робити:** Попросіть учнів по черзі додати ці правила і спостерігати, який колір перемагає:

```css
h1             { color: green; }    /* специфічність: 1   */
.header-title  { color: orange; }   /* специфічність: 10  */
#site-title    { color: purple; }   /* специфічність: 100 */
```

**Відповідь:** Переможе `#site-title { color: purple }` — найвища специфічність.

**Що обговорити:**
- Таблиця специфічності: `*`(0) < тег(1) < клас(10) < ID(100).
- Порядок у файлі важливий лише якщо специфічність **однакова** — тоді останнє правило перемагає.

## Типові помилки учнів

1. **Пробіл у ланцюжку** — `h2 .featured` замість `h2.featured`. Пробіл означає «нащадок», а не «і».
2. **Забутий `#` або `.`** — `site-title` замість `#site-title` не спрацює.
3. **Кілька ID на сторінці** — `id` має бути унікальним; якщо два елементи мають однаковий ID, це помилка HTML.
4. **Очікування hover на мобільних** — `:hover` не спрацьовує на сенсорних екранах без додаткових налаштувань.
