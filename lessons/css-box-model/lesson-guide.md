# CSS: Блокова модель — Конспект уроку (тільки для вчителя)

## Огляд

Учні працюють з файлами `index.html` + `style.css`. Сторінка вже структурована, але виглядає «зламаною». Кожне завдання додає одну CSS-властивість і одразу дає видимий результат.

**Теми (у порядку з Notion):**

1. Height & Width — висота та ширина
2. Border — рамка
3. Border Radius — закруглення рамки
4. Padding — внутрішній відступ
5. Margin — зовнішній відступ (окремі сторони → скорочений запис → `0 auto` → злиття)
6. Min / Max Width & Height — мінімальна та максимальна висота і ширина
7. Overflow — переповнення
8. Resetting Defaults — скидання стандартних стилів
9. Visibility — видимість

---

## Завдання 1 — Height & Width (висота та ширина)

**Концепція:** Властивості `width` і `height` визначають розмір області вмісту елемента. Пікселі задають точний фіксований розмір на будь-якому пристрої.

**Що учні бачать до:** Банер-герой видимий, але низький — лише 200px. Зображення космосу майже не видно.
**Що учні бачать після:** Великий, ефектний банер із повним зображенням космосу.

**CSS, який потрібно змінити у `.hero-banner`:**

```css
.hero-banner {
  height: 400px; /* було 200px */
}
```

**Що обговорити:**

- `height: 200px` проти `height: 400px` — покажіть, як зміна одного значення одразу змінює макет.
- `width: 100%` означає «займи всю ширину батьківського елемента». `<div>` вже робить це за замовчуванням як блоковий елемент.
- Пікселі дають точний розмір на будь-якому пристрої. Обговоріть компроміс: `px` передбачуваний, але не адаптивний. Повернемося до цього у Завданні 6 з `min-height` / `max-height`.
- Запитайте: *Що станеться, якщо повністю прибрати `height`?* (Блок стискається до 0px, бо немає текстового вмісту — стає невидимим.)

**Питання для обговорення:** Що станеться, якщо задати `height: 50%`? *(Браузер не може обчислити 50% від невідомої висоти батьківського елемента — нічого не зміниться, якщо батько не має фіксованої висоти.)*

## Завдання 2 — Border (рамка)

**Концепція:** `border` малює лінію навколо елемента. Потребує трьох значень: `ширина стиль колір`.

**Що учні бачать до:** Картка статті невидима — біла на білому фоні, межі не видно.
**Що учні бачать після:** Чітка, видима картка з обрамленням.

**CSS, який потрібно додати у `.article-preview`:**

```css
.article-preview {
  border: 2px solid #0f3460;
}
```

**Що обговорити:**

- Скорочений запис: `border: <ширина> <стиль> <колір>` — всі три значення обов'язкові для відображення рамки.
- Поширені значення `style`: `solid`, `dashed`, `dotted`, `none`.
- Без `solid` (або будь-якого стилю) рамка невидима, навіть якщо задано ширину та колір.
- Запитайте: *Яке значення `border` за замовчуванням?* (`border: medium none currentColor` — фактично невидима.)

**Спробуйте з учнями:** Змініть `solid` на `dashed` або `dotted`, щоб побачити різницю.

## Завдання 3 — Border Radius (закруглення рамки)

**Концепція:** `border-radius` закруглює кути елемента. Значення `50%` на квадратному елементі створює ідеальне коло.

**Що учні бачать до:** Картка статті має гострі кути 90°; аватар автора — квадрат.
**Що учні бачать після:** Закруглені кути картки; круглий аватар.

**CSS, який потрібно додати:**

```css
/* Частина А — закруглити кути картки */
.article-preview {
  border-radius: 8px;
}

/* Частина Б — зробити аватар колом */
.author-avatar {
  border-radius: 50%;
}
```

**Що обговорити:**

- `border-radius: 8px` — відстань від кута до початку заокруглення. Більше значення = більш округлі кути.
- `border-radius: 50%` — ідеальне коло **лише якщо ширина === висота**. Аватар вже 80×80px. Спробуйте задати різні width/height, щоб показати, як він стає еліпсом.
- Запитайте: *Що зробить `border-radius: 0`?* (Гострі кути — значення за замовчуванням.)

**Спробуйте:** Змінюйте `border-radius` від `8px` → `24px` → `50%` на картці статті, щоб побачити весь діапазон.

## Завдання 4 — Padding (внутрішній відступ)

**Концепція:** Padding — це простір **всередині** елемента між вмістом і рамкою.

**Що учні бачать до:** Обидві картки мають рамки, але текст впритул до краю — виглядає стиснуто і непрофесійно.
**Що учні бачать після:** Обидві картки мають зручний простір всередині.

**Крок 1 — додати padding до `.article-preview`** (щойно додали рамку у Завданні 2, тому стиснутий текст ще свіжий у пам'яті):

```css
.article-preview {
  padding: 20px;
}
```

**Крок 2 — повторити на `.fact-card`** і використати для навчання окремих сторін та скороченого запису:

```css
.fact-card {
  padding: 20px;
}
```

**Потім показати окремі сторони на `.fact-card`:**

```css
/* Замість padding: 20px спробуйте: */
.fact-card {
  padding-top: 16px;
  padding-right: 20px;
  padding-bottom: 16px;
  padding-left: 20px;
}

/* Той самий результат скороченим записом (top right bottom left — за годинниковою стрілкою): */
.fact-card {
  padding: 16px 20px 16px 20px;
}

/* Два значення (top/bottom  left/right): */
.fact-card {
  padding: 16px 20px;
}
```

**Що обговорити:**

- Padding використовує колір фону елемента — він знаходиться **всередині блоку**.
- Порядок скороченого запису: **top → right → bottom → left** (за годинниковою стрілкою).
- Два значення: перше = top/bottom, друге = left/right.
- Три значення: перше = top, друге = left/right, третє = bottom.

**Ключова відмінність, яку потрібно повторити:** *Padding — всередині рамки. Margin — зовні рамки.*

## Завдання 5 — Margin (зовнішній відступ)

### 5а — Відступи між абзацами через `margin-top`

**Концепція:** Margin — це простір **зовні** елемента, який відштовхує сусідні елементи.

**Що учні бачать до:** Чотири абзаци стоять впритул без жодного проміжку.
**Що учні бачать після:** Між абзацами з'являються чіткі проміжки.

**CSS, який потрібно додати:**

```css
.article-body p {
  margin-top: 24px;
}
```

**Що обговорити:**

- Покажіть ту саму логіку окремих сторін і скороченого запису, що й для padding.
- Запитайте: *Де з'являється простір біля першого абзацу?* (Зверху — він відштовхується від заголовку секції.)

---

### 5б — Центрування через `margin: 0 auto`

**Концепція:** Встановлення горизонтальних margin у `auto` наказує браузеру рівномірно розподілити залишковий простір з обох сторін — центруючи елемент.

---

**Частина 1 — конкретний елемент: банер розсилки**

**Що учні бачать до:** Банер розсилки (500px шириною) прилип до лівого краю.
**Що учні бачать після:** Банер ідеально відцентрований на сторінці.

**CSS, який потрібно додати:**

```css
.newsletter-banner {
  margin: 24px auto;
  padding: 14px 24px;
}
```

**Що обговорити:**

- `margin: 24px auto` → 24px зверху/знизу, **auto** зліва/справа.
- Auto margin центрує лише блокові елементи з **явно заданою шириною**. Запитайте учнів: *Що станеться, якщо прибрати `width: 500px` у `.newsletter-banner`?* (Елемент розтягнеться на 100% — залишкового простору немає — центрування не спрацює.)

---

**Частина 2 — контейнер сторінки: `.container`**

**Концепція:** Це той самий прийом, але застосований до всього вмісту сторінки. У реальних проєктах майже завжди є `.container` або `.wrapper`, який обмежує ширину сторінки і центрує весь вміст — щоб на широких моніторах текст не розтягувався від краю до краю.

**Що учні бачать до:** Весь вміст займає повну ширину сторінки — незручно читати на широких екранах.
**Що учні бачать після:** Вміст відцентровано і обмежено зручною шириною.

**CSS, який потрібно додати:**

```css
.container {
  max-width: 960px;
  margin: 0 auto;
}
```

**Що обговорити:**

- Це стандартний патерн у веб-розробці — він є в кожному реальному сайті.
- `max-width: 960px` — на малих екранах контейнер буде вужчим (100% ширини), на широких — не більше 960px.
- `margin: 0 auto` — центрує контейнер у межах `<main>`.
- Запитайте: *Де ще на сайтах використовують цей прийом?* (Шапка, футер, навігація — скрізь, де вміст має бути по центру і не розтягуватись на весь екран.)
- Покажіть різницю між `width: 960px` (фіксована — зламається на малих екранах) і `max-width: 960px` (адаптивна — завжди коректна).

---

### 5в — Злиття відступів (обговорення, зміни в коді не потрібні)

**Концепція:** Горизонтальні margin завжди додаються. Вертикальні (top + bottom) **зливаються** — перемагає більше значення.

**Демонстрація:**

```css
/* Два елементи один під одним: */
#element-a {
  margin-bottom: 10px;
}
#element-c {
  margin-top: 30px;
}
/* Фактичний проміжок = 30px (НЕ 40px) */

/* Два елементи поруч: */
#element-a {
  margin-right: 10px;
}
#element-b {
  margin-left: 10px;
}
/* Фактичний проміжок = 20px (10 + 10 — додаються) */
```

**Що обговорити:**

- Це одна з найпоширеніших несподіванок у CSS.
- Запитайте: *Навіщо браузер так робить?* (Щоб уникнути подвійних проміжків між абзацами, у яких є і `margin-bottom`, і `margin-top`.)
- Padding ніколи не зливається — завжди додається повністю.

## Завдання 6 — Min / Max Width & Height (мін./макс. ширина та висота)

**Концепція:** `min-width` / `max-width` / `min-height` / `max-height` задають межі, щоб елементи адаптувалися до різних розмірів екрана без порушення макету.

**Що учні бачать до:** Цитата розтягується на весь екран — довгі рядки, важко читати.
**Що учні бачать після:** Цитата обмежена зручною шириною незалежно від розміру екрана.

**CSS, який потрібно додати у `.featured-quote`:**

```css
.featured-quote {
  max-width: 680px;
  min-height: 80px;
}
```

**Що обговорити:**

- `max-width: 680px` — елемент може бути вужчим (на малих екранах), але ніколи не ширшим за 680px.
- `min-height: 80px` — елемент завжди щонайменше 80px заввишки, навіть якщо вміст короткий.
- ⚠️ Увага: якщо `max-height` задати занадто малим, вміст «вилізе» за межі — це веде до Завдання 7!
- Попросіть учнів змінити розмір вікна браузера, щоб побачити `max-width` у дії.

**Спробуйте:** Додайте `max-height: 60px` і подивіться, як текст виходить за межі блоку цитати — ідеальний місток до Завдання 7.

## Завдання 7 — Overflow (переповнення)

**Концепція:** Коли вміст більший за контейнер (через фіксовану `height` або `max-height`), властивість `overflow` визначає, що робити з надлишковим вмістом.

**Що учні бачать до:** Журнал місії має висоту 120px, але містить більше тексту — він «вилізає» за межі блоку.
**Що учні бачають після (кожен варіант):**

- `overflow: hidden` → надлишковий вміст обрізається (невидимий)
- `overflow: scroll` → з'являється смуга прокрутки, весь вміст доступний
- `overflow: visible` → поведінка за замовчуванням, вміст виходить за межі (безлад)

**CSS, який потрібно додати у `.mission-log`:**

```css
.mission-log {
  overflow: scroll;
}
```

**Що обговорити:**

- Спробуйте всі три значення, щоб учні побачили кожен результат.
- Згадайте `overflow-x` (горизонтальний) і `overflow-y` (вертикальний) для точного керування.
- `overflow` задається на **батьківському контейнері**, а не на вмісті, що виходить за межі.
- Найпрактичніший вибір: `overflow: hidden` (обрізати) або `overflow: scroll` (дати змогу прокручувати).

**Питання для обговорення:** *Коли б ви використали `overflow: hidden`?* (Обрізання зображень у круглому аватарі, приховування випадаючого меню за межами навігації тощо.)

## Завдання 8 — Resetting Defaults (скидання стандартних стилів)

**Концепція:** Кожен браузер має вбудовану таблицю стилів (user agent stylesheet), яка додає стандартні `margin` і `padding` до елементів — `<h1>`, `<p>`, `<body>` — все має стандартні відступи браузера. Розробники часто скидають їх, щоб почати з чистого аркуша.

**Що учні бачать до:** Заголовки та абзаци мають стандартні відступи браузера (трохи відрізняються у Chrome і Firefox).
**Що учні бачать після:** Всі стандартні margin і padding прибрані глобально.

**Дія:** Розкоментувати блок скидання на самому початку `style.css`:

```css
* {
  margin: 0;
  padding: 0;
}
```

**Що обговорити:**

- `*` вибирає **кожен елемент** на сторінці.
- `margin: 0` і `padding: 0` — значення `0` не потребує одиниці виміру (px, em тощо).
- Зверніть увагу: сторінка трохи зміниться (зникне відступ body, зміниться відстань між заголовками), але явно записані нами правила (наприклад, `margin-top: 24px` для абзаців) **не постраждають** — вони перевизначають скидання, бо є більш специфічними.
- Багато реальних проєктів використовують розширений варіант:
  ```css
  *,
  *::before,
  *::after {
    box-sizing: border-box;
    margin: 0;
    padding: 0;
  }
  ```

## Завдання 9 — Visibility (видимість)

**Концепція:** `visibility: hidden` приховує елемент візуально, але залишає його місце в макеті. `display: none` повністю видаляє елемент — місця не резервується.

**Що учні бачать до:** «Premium» видно в навігаційному меню.
**Що учні бачать після:** «Premium» зникає, але порожнє місце залишається.

**CSS, який потрібно додати у `.nav-item--soon`:**

```css
.nav-item--soon {
  visibility: hidden;
}
```

**Потім порівняти з `display: none`:**

```css
.nav-item--soon {
  display: none; /* елемент повністю видаляється, місця не займає */
}
```

**Таблиця порівняння:**

| Властивість           | Видимий? | Займає місце? |
| --------------------- | -------- | ------------- |
| `visibility: visible` | Так      | Так           |
| `visibility: hidden`  | Ні       | **Так**       |
| `display: none`       | Ні       | **Ні**        |

**Що обговорити:**

- Запитайте: *Коли використовувати `visibility: hidden` замість `display: none`?* (Коли потрібно анімувати появу/зникнення елемента без стрибків макету, або коли потрібно зарезервувати місце, поки елемент неактивний.)
- Значення `collapse` існує для рядків таблиць — згортає рядок без зміщення таблиці.

| 9        | `visibility: hidden` проти `display: none` | Пункт меню ховається з місцем або без                    |

## Типові помилки учнів

1. **Забута одиниця виміру** — `margin: 24` замість `margin: 24px`. CSS мовчки ігнорує невалідні значення.
2. **Забутий стиль рамки** — `border: 2px #333` без `solid` нічого не відображає.
3. **`margin: auto` не працює** — у елемента немає явно заданої `width`, тому нема що центрувати.
4. **Padding не видно на inline-елементах** — використовуйте `display: inline-block` або `display: block`.
5. **Плутанина padding і margin** — повторюйте: *padding — всередині рамки, margin — зовні*.
6. **Очікування прокрутки від `max-height`** — він лише обрізає; для прокрутки потрібен `overflow: scroll`.
